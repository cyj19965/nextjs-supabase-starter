# PRD: 오늘의 뜨개 (My Knitting Counter)

## 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | 오늘의 뜨개 |
| 한 줄 소개 | 뜨개질 작품별 **단수 카운터**와 진행률을 관리하는 뜨개인 전용 트래커 |
| 목적 | 뜨개질하다 "지금 몇 단째더라?"를 해결 — 드라마 보면서 떠도 +1 버튼만 누르면 단수가 기록되고, 작품이 몇 % 진행됐는지 한눈에 보임 |
| 대상 사용자 | 뜨개인 (가입한 사용자 각자 자기 작품만 관리) |

### Supabase를 선택한 이유

- **인증 내장**: 이메일 로그인으로 폰/태블릿 어디서든 내 카운터에 접근
- **Postgres + RLS**: "내 작품은 나만 본다"를 DB 정책으로 보장
- **빠른 업데이트**: 카운터 +1은 단순 update라 Supabase로 충분히 즉각적

## 2. 핵심 기능

1. **회원가입/로그인** — Supabase Auth (이메일)
2. **작품(프로젝트) CRUD** — 작품명, 종류, 실 정보, 바늘 호수, 목표 단수, 상태
3. **단수 카운터** ⭐ 핵심 — 작품 상세에서 큰 **+1 버튼** (실수 복구용 −1 포함), 현재 단수 저장
4. **진행률 바** — 현재 단수 ÷ 목표 단수 %, 목록 카드에도 표시
5. **상태 관리** — 진행중 / 완성 🎉 / 잠듦(방치) — 목록에서 진행중 우선 정렬
6. **패턴 메모** — 도안 요약, 코 줄이기 위치 같은 노트

## 3. 기술 스택

- Next.js 15 (App Router) + TypeScript
- Supabase (Auth, Postgres, RLS)
- Tailwind CSS + shadcn/ui (템플릿 내장)

## 4. 데이터 모델 (초안)

`projects` 테이블:

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | uuid (pk) | 기본키 |
| user_id | uuid (fk → auth.users) | 작성자 — RLS 기준 |
| name | text, not null | 작품명 (예: 엄마 목도리) |
| kind | text | 목도리/모자/가디건/장갑/인형/가방/기타 |
| yarn | text | 실 정보 (예: 램스울 앤디미 3볼) |
| needle | text | 바늘 (예: 대바늘 4.5mm) |
| goal_rows | int | 목표 단수 (없으면 null — 진행률 미표시) |
| current_rows | int, default 0, >= 0 | 현재 단수 ⭐ 카운터 대상 |
| status | text, default '진행중' | 진행중/완성/잠듦 |
| started_at | date | 시작일 |
| memo | text | 패턴 노트 |
| created_at | timestamptz | 생성 시각 |

**RLS**: 모든 작업(select/insert/update/delete)을 `auth.uid() = user_id`로 제한.

## 5. 사용자 스토리

- 뜨개인으로서, 드라마 보면서 뜨다가도 +1 버튼만 눌러 단수를 놓치지 않는다
- 뜨개인으로서, 목도리가 68% 진행된 걸 보고 완성까지 동기를 얻는다
- 뜨개인으로서, 반년째 잠들어 있는 가디건을 목록에서 발견하고 다시 꺼낸다
- 사용자로서, 내 작품 목록은 나에게만 보인다 (RLS)

## 6. MVP 범위

**포함**: 인증(템플릿 내장), projects 테이블 + RLS, 작품 CRUD, 단수 카운터(+1/−1/직접 입력), 진행률 바, 상태별 정렬
**제외(Phase 2)**: 완성작 사진 갤러리(Storage), 실 재고장, 폰-태블릿 실시간 동기화(Realtime), 패턴 파일 첨부
