# PRD: 우리동네 맛집 기록장

## 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | 우리동네 맛집 기록장 |
| 한 줄 소개 | 가본 맛집을 별점과 한줄평으로 기록하고, 다음에 "거기 어디였지?"를 바로 찾는 나만의 맛집 지도 |
| 목적 | 방문한 식당/카페의 기억(맛, 메뉴, 재방문 의사)이 휘발되는 문제 해결 — 로그인해서 어디서든 기록·검색 |
| 대상 사용자 | 본인 (가입한 사용자 각자 자기만의 기록장을 가짐) |

### Supabase를 선택한 이유

- **인증 내장**: 이메일 로그인/회원가입을 직접 구현하지 않아도 됨
- **Postgres + RLS**: "내 기록은 나만 본다"를 DB 정책(Row Level Security)으로 보장
- **무료 티어**: 개인 프로젝트에 충분

## 2. 핵심 기능

1. **회원가입/로그인** — Supabase Auth (이메일), 로그인해야 기록 가능
2. **맛집 기록 CRUD** — 가게명, 카테고리, 별점(1~5), 한줄평, 방문일, 재방문 의사
3. **목록/상세** — 내 기록을 최신순 카드로, 상세에서 전체 내용
4. **필터/정렬** — 카테고리별, 별점순
5. **검색** — 가게명으로 검색

## 3. 기술 스택

- Next.js 15 (App Router) + TypeScript
- Supabase (Auth, Postgres, RLS)
- Tailwind CSS + shadcn/ui (템플릿 내장)

## 4. 데이터 모델 (초안)

`places` 테이블:

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| id | uuid (pk) | 기본키 |
| user_id | uuid (fk → auth.users) | 작성자 — RLS 기준 |
| name | text, not null | 가게명 |
| category | text | 한식/중식/일식/양식/카페/디저트/기타 |
| rating | int (1~5) | 별점 |
| comment | text | 한줄평 |
| visited_at | date | 방문일 |
| revisit | boolean | 재방문 의사 |
| created_at | timestamptz | 생성 시각 |

**RLS**: 모든 작업(select/insert/update/delete)을 `auth.uid() = user_id`로 제한.

## 5. 사용자 스토리

- 회원가입한 사용자로서, 오늘 간 식당을 별점과 함께 기록해서 나중에 다시 찾아볼 수 있다
- 로그인한 사용자로서, "한식" 카테고리만 별점순으로 모아 봐서 오늘 저녁 갈 곳을 고를 수 있다
- 사용자로서, 다른 사람의 기록은 보이지 않고 내 기록만 보인다 (RLS)

## 6. MVP 범위

**포함**: 인증(템플릿 내장), places 테이블 + RLS, 기록 작성/목록/상세, 카테고리 필터
**제외(Phase 2)**: 사진 업로드(Storage), 지도 표시, 검색, 공유 링크
