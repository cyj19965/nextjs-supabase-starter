# ROADMAP: 오늘의 뜨개

[PRD](PRD.md) 구현 순서. **골격 → 공통 → 핵심 → 확장** 원칙을 따른다.

## Phase 1: MVP (예상 4~7일)

### 1-1. 골격 (완료된 것 + 남은 것)

- [x] Next.js + Supabase 템플릿 생성, Supabase 프로젝트 연결 (미션 21)
- [x] 개발 환경 설정 — 커맨드/훅/에이전트/MCP/docs (미션 22)
- [ ] 템플릿 기본 페이지 정리 (튜토리얼 컴포넌트 제거, 홈 골격)

### 1-2. 데이터 계층 (공통)

- [ ] `projects` 테이블 마이그레이션 + **RLS 정책** (`/db-table` 커맨드 활용) — `current_rows >= 0` 체크 제약 포함
- [ ] `Project` 타입 정의 및 쿼리 함수 (`lib/projects.ts`) — Supabase 호출을 여기로 격리
- [ ] rls-checker 에이전트로 정책 검증

### 1-3. 핵심 기능

- [ ] 작품 등록 폼 (`/projects/new`) — server action으로 insert
- [ ] 내 작품 목록 (`/projects`) — 진행중 우선, 진행률 바 카드
- [ ] 작품 상세 (`/projects/[id]`) — **단수 카운터 (+1 / −1 / 직접 입력)** ⭐ 이 서비스의 존재 이유
- [ ] 상태 변경 (진행중/완성/잠듦) 및 수정/삭제

### Phase 1 완료 기준

- [ ] 회원가입 → 작품 등록 → +1 연타 → 진행률 상승 → 완성 처리 전 과정이 동작
- [ ] −1이 0 아래로 내려가지 않는다 (경계 처리)
- [ ] 다른 계정으로 로그인하면 남의 작품이 보이지 않음 (RLS 검증)
- [ ] `npm run build` 통과, Vercel 배포

## Phase 2: 기능 확장 (MVP 검증 후)

- [ ] 완성작 사진 갤러리 (Supabase Storage)
- [ ] 실 재고장 (yarns 테이블 — 남은 볼 수 관리)
- [ ] 폰-태블릿 실시간 카운터 동기화 (Supabase Realtime)
- [ ] 키보드 스페이스바 = +1 (핸즈프리에 가깝게)
- [ ] 완성 작품 아카이브 페이지

## 원칙

- 한 번에 한 기능, 기능마다 커밋 (미션 20 방식)
- 테이블 추가/변경 시 RLS부터 생각한다
- 카운터는 이 앱의 심장 — 반드시 실제로 눌러보고(버튼 → DB 반영 확인) 커밋한다
- MVP 완료 전 Phase 2 항목에 손대지 않는다
