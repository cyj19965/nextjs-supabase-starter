# ROADMAP: 오늘의 뜨개

[PRD](PRD.md) 구현 순서. **골격 → 공통 → 핵심 → 확장** 원칙을 따른다.

## Phase 1: MVP (예상 4~7일)

### 1-1. 골격 (완료된 것 + 남은 것)

- [x] Next.js + Supabase 템플릿 생성, Supabase 프로젝트 연결 (미션 21)
- [x] 개발 환경 설정 — 커맨드/훅/에이전트/MCP/docs (미션 22)
- [x] 템플릿 기본 페이지 정리 (홈을 뜨개 랜딩으로 교체)

### 1-2. 데이터 계층 (공통)

- [x] `projects` 테이블 마이그레이션 + **RLS 정책** — `current_rows >= 0` 체크 제약 포함, 원격 DB 적용 완료
- [x] `Project` 타입 정의 및 쿼리 함수 (`lib/projects.ts`) — Supabase 호출을 여기로 격리
- [x] RLS 검증 — anon key 0건, 교차 사용자 테스트(user2가 user1 작품 못 봄) 통과

### 1-3. 핵심 기능

- [x] 작품 등록 폼 (`/projects/new`) — server action으로 insert
- [x] 내 작품 목록 (`/projects`) — 진행중 우선, 진행률 바 카드
- [x] 작품 상세 (`/projects/[id]`) — **단수 카운터 (+1 / −1 / 직접 입력)** ⭐
- [x] 상태 변경 (진행중/완성/잠듦) 및 삭제

### Phase 1 완료 기준

- [x] 로그인 → 작품 등록 → +1 연타 → DB 반영 전 과정 동작 (실제 브라우저로 검증)
- [x] −1이 0 아래로 내려가지 않는다 (서버 액션에서 clamp + DB check 제약 이중 방어)
- [x] 다른 계정으로 로그인하면 남의 작품이 보이지 않음 (RLS 검증)
- [x] `npm run build` 통과
- [ ] Vercel 배포

## 원칙

- 한 번에 한 기능, 기능마다 커밋 (미션 20 방식)
- 테이블 추가/변경 시 RLS부터 생각한다
- 카운터는 이 앱의 심장 — 반드시 실제로 눌러보고(버튼 → DB 반영 확인) 커밋한다
- MVP 완료 전 Phase 2 항목에 손대지 않는다
